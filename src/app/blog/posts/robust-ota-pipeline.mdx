Firmware delivery is easy. **Recovery** is the hard part.

## Non-negotiables

1. **Two slots + A/B marker** with a _monotonic_ version.
2. **Rollout windows**: maintenance vs fleet hours.
3. **Staged reboot**: device reports `READY_TO_REBOOT`, cloud schedules.
4. **Downgrade gate**: only for rescue builds signed by a separate key.

I use a tiny state machine that logs each leg (`downloaded → verified → staged → rebooted → running`), which pays off in field triage.
